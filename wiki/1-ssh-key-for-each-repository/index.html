1-ssh-key-for-each-repository

Usando uma chave de SSH para cada repositório do GitHub

Útil para projetos com acesso a submódulos restrito

  1. Criar as ssh keys
    Para cada repositório, repita o processo abaixo:

    # o campo de e-mail é, na verdade, um campo para identificador da credencial. Pode ser qualquer nome distinguivel entre os repositórios (e.g. "gabrielmelocomp@nome_do_repo")

ssh-keygen -t ed25519 -C "email@email.com"

# altere o nome do arquivo da chave para conter algum identificador do repositorio

# inicia o agente ssh
eval "$(ssh-agent -s)"

# adiciona a key 
ssh-add ~/.ssh/nome_do_repositorio

  2. Incluir as keys como deploy-keys no GitHub em cada repositório

  3. Criar o arquivo ~/.ssh/config para manusear como o uso das chaves em cada repositório

Host repo_1
        HostName github.com
        User git
        IdentityFile ~/.ssh/nome_repo_1
        IdentitiesOnly yes

Host repo_2
        HostName github.com
        User git
        IdentityFile ~/.ssh/nome_repo_2
        IdentitiesOnly yes

Extras

  • Para aumentar o log dos comandos ssh executados pelo git:

    export GIT_SSH_COMMAND='ssh -v'

  • Para realizar push dos submodules automaticamente, quando der push do repo raiz:

    git config push.recurseSubmodules on-demand

  • Para realizar pull dos submodules automaticamente, quando der pull do repo raiz:

    git config --global submodule.recurse true
